# מערכת ניהול לידים למשרות

מערכת Web בעברית מלאה (RTL) לניהול פונים למשרות בק אופיס, שירות לקוחות טלפוני ומוקדי תמיכה טכנית.

## הפעלה מקומית

```bash
npm install
npm run dev
```

המערכת תרוץ ב-`http://localhost:5173`.

## מבנה

- `/` – דף נחיתה ציבורי עם טופס השארת פרטים.
- `/login` – כניסת מנהל.
- `/admin` – דשבורד פנימי (מוגן בהתחברות).
- `/admin/leads` – טבלת פונים, חיפוש וסינון.
- `/admin/leads/:id` – כרטיס פונה: סטטוס, הערות, היסטוריה, כפתורי WhatsApp.
- `/admin/reminders` – תזכורות לבדיקה אחרי חודשיים.
- `/admin/areas` – ניהול אזורים/אתרים.
- `/admin/templates` – עריכת תבניות WhatsApp.
- `/admin/account` – החלפת סיסמה.

## אבטחה

- סיסמאות נשמרות בלוקאל סטוראג' כ-hash בלבד (SHA-256 דרך SubtleCrypto).
- כל הפעולות הניהוליות מוגנות מאחורי `ProtectedRoute`.
- ולידציה גם בצד לקוח (טלפון נייד ישראלי, ת.ז עם בדיקת ספרת ביקורת).
- שדות טקסט חופשי עוברים סניטציה בסיסית למניעת XSS.

## וואטסאפ

הכפתורים בכרטיס הפונה פותחים `wa.me` עם הודעה מוכנה. אין שליחה אוטומטית – המשתמש מאשר ידנית ב-WhatsApp.

## מעבר ל-Supabase + Netlify (בעתיד)

הנתונים מאוחסנים כרגע ב-`localStorage` תחת מפתחות `jls.*`. כדי להעביר:

1. החליפו את הפונקציות ב-`src/lib/storage.js` בקריאות `supabase.from('leads')...` וכו'.
2. החליפו את `AuthContext` להשתמש ב-`supabase.auth`.
3. ב-Netlify: build command = `npm run build`, publish directory = `dist`.

המבנה של הקונטקסטים (`useData`, `useAuth`) זהה – הקומפוננטות לא יידרשו שינוי.
